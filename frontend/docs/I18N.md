# I18N

## Поддерживаемые языки

Frontend поддерживает две локали:

- `en` - английский, локаль по умолчанию и fallback.
- `ru` - русский.

Источник базового контракта - пакет `packages/i18n` (`@airlinesim/i18n`). Он хранит список поддерживаемых локалей, ключ `localStorage`, нормализацию browser locale и helper `translate`.

## Где хранить строки

Строки хранятся рядом с приложением или feature, которая ими владеет. Для shell локали разделены по файлам:

- `apps/shell/src/i18n/en.ts` - английские строки.
- `apps/shell/src/i18n/ru.ts` - русские строки.
- `apps/shell/src/i18n/messages.ts` - сборка локалей и экспорт типа ключей.

Не нужно складывать все строки всех MFE в один глобальный файл. Remote-приложение должно иметь свой локальный словарь и использовать общие типы/helper из `@airlinesim/i18n`.

Правило добавления строки:

1. Добавить один стабильный ключ в файл `en.ts`.
2. Добавить тот же ключ в файл `ru.ts`.
3. Использовать `translate(messages, locale, key)`.
4. Если строка нужна нескольким приложениям, вынести только общий ключ/словарь в shared package после явной необходимости.

## Shell

Shell владеет выбранной локалью:

- читает `airlinesim:locale` из `localStorage`;
- если сохраненной локали нет, нормализует `navigator.language`;
- применяет `document.documentElement.lang`;
- сохраняет новое значение в `localStorage`;
- эмитит `i18n:locale-changed` через `@airlinesim/event-bus`;
- передает `appLocale` в Vue remotes и Svelte map через Module Federation props.

Переключатель языка находится в topbar. Переключение работает между `en` и `ru`.

## MFE-контракт

Каждое remote-приложение должно принимать prop:

```ts
appLocale?: "en" | "ru";
```

Shell передает этот prop при рендере remote. Для реактивного обновления без remount remote может дополнительно подписаться на event-bus:

```ts
import { airlineSimEventBus } from "@airlinesim/event-bus";

const unsubscribe = airlineSimEventBus.on("i18n:locale-changed", ({ locale }) => {
  // обновить локальное состояние remote
});
```

Для Vue remote обычно достаточно `watch(() => props.appLocale, ...)`. Для Svelte remote props обновляются через wrapper, поэтому `$props()` должен включать нужную локаль, если remote использует локальные строки.

## Fallback

Fallback всегда `en`. Если browser locale не `ru`, shell выбирает `en`. Если ключ отсутствует в выбранной локали, `translate` возвращает английское значение, а затем сам ключ.

## Проверки

После добавления или изменения строк запускать:

```bash
bun run lint
```

из директории `frontend`.
